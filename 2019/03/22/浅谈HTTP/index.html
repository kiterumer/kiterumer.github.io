HTTP是什么呢？

学名超文本传输协议（HyperText Transfer Protocol），用于传输诸如HTML的超媒体文档的应用层协议。
通俗点就是指导客户端和服务器如何沟通的一种协议，属于TCP/IP协议的一部分。


客户端发送请求，服务端收到请求后返回响应，那么请求和响应都有哪些秘密呢？

请求的格式

  • 请求最多包含四部分，最少包含三部分。（也就是说第四部分可以为空）
  • 第三部分永远都是一个回车（\n）
  • 动词有 GET POST PUT PATCH DELETE HEAD OPTIONS 等
  • 这里的路径包括「查询参数」，但不包括「锚点」
  • 如果没有写路径，那么路径默认为 /
  • 第 2 部分中的 Content-Type 标注了第 4 部分的格式

补充

请求的方法有很多，比如GET POST PUT PATCH DELETE HEAD OPTIONS 等

  • GET 从服务器获取资源,简单但安全性不高
  • POST 向服务器发送数据
  • PUT 全部修改
  • PATCH 部分修改
  • DELETE 删除

响应的格式

两者形式上比较相似
中间的部分都是以键值对的形式书写的,空行不能丢

举个栗子

地址栏输入www.baidu.cim 点击回车，然后右键鼠标选择检查进入喜闻乐见的开发者界面，默认显示的Elements选项，我们选择Network选项，即可查看访问百度网站期间发生了写什么。如果什么也没有刷新一下，于是许许多多的请求从上到下一次罗列了出来


点击其中某个请求比如第一个，会进入如下界面，详细地显示了请求内容和响应内容


点击view source可看到标准格式的请求和响应

利用curl发送请求

打开Git Bash控制台，输入curl -s -v -H "yyy:abc" -- "https://www.baidu.com" 回车


大于号对应的即是请求体，小于号对应的即是响应体
具体的参数诸如-s,-v,-H可参考这个神奇的网站，直接拷贝搜索，会有详细的解释，前提你得会点英语
